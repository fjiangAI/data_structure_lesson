# Week 02 线性表之一：顺序表 教师教案

## 本周教学意图

本周不是让学生记住 顺序表 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：连续的线

它是最接近硬件内存的线性结构，随机访问强，但中间插入删除要搬移元素。

## 与前后知识的连接

- 承接 Week 01 算法分析：本周不急着进入某个结构，而是让学生先看到：程序中的对象必须被组织起来，操作才有可分析的边界。
- 导向 Week 03 链表：链表把线性关系从物理相邻转移到指针连接，结点可以散落在内存中。

教师讲解时应不断提醒学生：结构复杂度增加，不是为了炫技，而是为了表达更复杂的关系或降低某些操作代价。

## 板书主线

```text
对象：线性结构、连续存储
关系：顺序表把元素排成连续线段，位置就是关系，地址计算就是访问能力。
不变量：顺序表把线性关系映射到连续内存，逻辑相邻通常也是物理相邻。它用容量 capacity 和长度 size 分离可用空间与已有元素。
操作：按位访问、尾部插入、中间插入
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：为什么连续性既带来 O(1) 按位访问，也带来 O(n) 移动代价？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. **按位访问**：直接计算地址 base + i * sizeof(ElemType)。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(1)。
2. **尾部插入**：容量足够时直接写入。分析时先找会重复执行的语句，再判断重复次数，所以代价为 均摊 O(1)。
3. **中间插入**：从后向前移动元素。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
4. **按值查找**：顺序扫描比较。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
5. **删除元素**：后续元素前移。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。

## 本周专属讲法

课堂上不要平均用力，建议抓住下面这条线：

1. 用数组 [10,20,30,40,_,_] 在下标 2 插入 25。
2. 先移动 40 到下标 4，再移动 30 到下标 3，最后写入 25 并让 size 从 4 变为 5。
3. 故意让学生从前往后移动一次，观察 30 被覆盖，从而理解移动方向不是代码细节，而是不变量要求。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 把 capacity 当成已有元素个数
- 插入移动方向写反
- 删除后忘记 size--
- 扩容后旧指针处理不清

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 顺序表中间插入时从后往前移动元素。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 顺序表 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 连续的线 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 顺序表 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能手推顺序表插入/删除的移动方向，并解释 `size/capacity` 不变量。
- 能指出一段模型生成代码中最可能出错的地方。
