# Week 16 综合设计、复习与 AI 辅助实践 教师教案

## 本周教学意图

本周不是让学生记住 综合复习 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：综合结构设计

综合设计的核心是维护多个结构之间的一致性，让每个操作都更新所有相关视图。

## 与前后知识的连接

- 承接 Week 15 线性排序：排序把比较、移动、稳定性和额外空间放在一起考察，是结构选择和操作代价的集中训练。
- 导向综合复习：能为真实问题组合多种结构，并维护一致性。

教师讲解时应不断提醒学生：结构复杂度增加，不是为了炫技，而是为了表达更复杂的关系或降低某些操作代价。

## 板书主线

```text
对象：结构选择、接口设计
关系：真实系统往往不是只用一种结构，而是组合顺序存储、索引、排序和图式关系。
不变量：综合设计要求先建模，再选择结构，最后实现操作。评价一个方案时同时看正确性、复杂度、可维护性和测试证据。
操作：需求建模、结构选择、接口实现
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：如果一个系统同时有顺序表和哈希索引，删除记录时最容易漏掉什么？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. **需求建模**：识别对象、关系和高频操作。这不是单个算法的复杂度结论，而是综合设计的评审环节，要求形成 对象-操作表。
2. **结构选择**：用访问模式决定数组、链表、树、图或哈希。这不是单个算法的复杂度结论，而是综合设计的评审环节，要求形成 结构取舍表。
3. **接口实现**：保持 ADT 边界清晰。这不是单个算法的复杂度结论，而是综合设计的评审环节，要求形成 头文件与不变量说明。
4. **复杂度评审**：给出关键操作的上界。这不是单个算法的复杂度结论，而是综合设计的评审环节，要求形成 操作复杂度矩阵。
5. **AI 辅助验证**：让模型生成反例、边界用例和解释。这不是单个算法的复杂度结论，而是综合设计的评审环节，要求形成 反例与修正记录。

## 本周专属讲法

课堂上不要平均用力，建议抓住下面这条线：

1. 学生记录系统可以用顺序表保存记录，用哈希表从学号定位顺序表下标，用排序生成成绩排名。
2. 插入时必须同时写顺序表和哈希索引；删除中间记录后，顺序表后续元素下标变化，索引必须重建或修正。
3. 综合设计不是问某个结构会不会写，而是问高频操作、结构职责、复杂度和一致性维护是否匹配。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 把多个结构堆在一起但不说明职责
- 删除记录后不更新索引
- 只报告平均复杂度不说明装载因子
- 把 LLM 生成代码当成系统设计

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 学生记录系统中顺序表、哈希索引和排名视图的一致性。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 综合复习 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 综合结构设计 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 综合复习 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能为一个小系统写出对象-操作-结构选择表，并说明每个操作维护哪些结构。
- 能指出一段模型生成代码中最可能出错的地方。
