# Week 09 堆与优先队列 教师教案

## 本周教学意图

本周不是让学生记住 堆 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：分叉的树

二叉树、BST、AVL 和堆都在控制树形关系：遍历回答“如何访问全部”，平衡回答“如何控制高度”，堆回答“如何快速取得最优”。

## 板书主线

```text
对象：完全二叉树、数组存储
关系：树把一条线扩展为层次结构，一个对象可以拥有多个后继，递归成为自然表达方式。
不变量：堆是一棵满足堆序性质的完全二叉树，常用数组存储。它只保证根是全局最优，不保证整体有序。
操作：push、pop、peek
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：树的高度为什么常常比结点总数更能决定操作代价？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. **push**：插入尾部后上滤。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(log n)。
2. **pop**：根与尾交换后下滤。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(log n)。
3. **peek**：读取根元素。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(1)。
4. **buildHeap**：从最后一个非叶结点开始下滤。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
5. **heapSort**：反复取堆顶。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n log n)。

## 本周专属讲法

课堂上不要平均用力，建议抓住下面这条线：

1. 数组下标 i 的父结点是 `(i-1)/2`，左右孩子是 `2*i+1` 和 `2*i+2`。
2. 最小堆插入 3 时先放末尾，再和父结点比较，逐层上滤到正确位置。
3. pop 时用末尾元素补根，再下滤，必须在左右孩子中选择更小者。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 把堆数组看成有序数组
- 下滤只比较左孩子
- pop 后忘记缩小 size
- 混淆大根堆和小根堆

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 堆的数组下标关系与上滤/下滤。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 堆 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 分叉的树 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 堆 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能在数组和完全二叉树之间转换，并手推 push/pop 的修复路径。
- 能指出一段模型生成代码中最可能出错的地方。
