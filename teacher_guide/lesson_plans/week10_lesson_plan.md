# Week 10 图的表示与遍历 教师教案

## 本周教学意图

本周不是让学生记住 图遍历 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：更复杂的图

图算法本质上是在关系网络中传播状态：visited、dist、prev、indegree 都是对关系进展的记录。

## 与前后知识的连接

- 承接 Week 09 堆：二叉树、BST、AVL 和堆都在控制树形关系：遍历回答“如何访问全部”，平衡回答“如何控制高度”，堆回答“如何快速取得最优”。
- 导向 Week 11 图算法：图取消了层次限制，用顶点和边描述任意多对多关系。

教师讲解时应不断提醒学生：结构复杂度增加，不是为了炫技，而是为了表达更复杂的关系或降低某些操作代价。

## 板书主线

```text
对象：图、顶点
关系：图取消了层次限制，用顶点和边描述任意多对多关系。
不变量：图用于描述任意对象之间的关系。邻接矩阵适合稠密图和快速判断边，邻接表适合稀疏图和遍历邻接点。
操作：添加边、判断边存在、DFS
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：为什么图算法一旦忘记 visited 或 dist 的含义，就很容易变成背步骤？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. 添加边：矩阵赋值或链表头插，复杂度 O(1)。
2. 判断边存在：矩阵直接读，邻接表扫描，复杂度 O(1) 或 O(deg)。
3. DFS：沿一条路径尽量深入，复杂度 O(V+E)。
4. BFS：按距离层次扩展，复杂度 O(V+E)。
5. 连通分量：多次从未访问顶点启动遍历，复杂度 O(V+E)。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 把结构名当成答案，不解释对象之间的关系。
- 只写正常输入路径，不写空、满、越界、重复或不存在的情况。
- 把复杂度结论背下来，却不能指出循环变量或递归规模。
- 看到 LLM 给出可编译代码就停止检查，不验证结构不变量。

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 添加边。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 图遍历 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 更复杂的图 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 图遍历 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能解释至少一个最坏情况。
- 能指出一段模型生成代码中最可能出错的地方。
