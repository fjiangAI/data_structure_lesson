# Week 15 排序算法之三：线性时间排序 教师教案

## 本周教学意图

本周不是让学生记住 线性排序 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：面向重排的算法

排序把比较、移动、稳定性和额外空间放在一起考察，是结构选择和操作代价的集中训练。

## 板书主线

```text
对象：比较排序下界、计数排序
关系：排序不是新结构，而是在数组或记录序列上重建顺序关系。
不变量：线性时间排序不是通用比较排序。它们利用关键字范围、位数或分布信息，把比较问题转化为计数和分配问题。
操作：计数排序、桶排序、基数排序
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：为什么没有一种排序在所有输入、空间、稳定性要求下都最优？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. **计数排序**：统计每个关键字出现次数并前缀累加。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n+k)。
2. **桶排序**：按分布把元素放入桶，再桶内排序。分析时先找会重复执行的语句，再判断重复次数，所以代价为 期望 O(n+k)。
3. **基数排序**：按位稳定排序多轮。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(d(n+r))。
4. **稳定分配**：从后向前放置可保持相等元素顺序。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
5. **范围压缩**：处理 min 到 max 的偏移。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n+k)。

## 本周专属讲法

课堂上不要平均用力，建议抓住下面这条线：

1. 计数排序对关键字范围 0..k 建立计数数组，代价是 O(n+k)。如果 k 远大于 n，空间和初始化成本会很高。
2. 稳定计数排序要先做前缀和，再从右向左扫描输入放入输出数组。
3. 基数排序每一位都要用稳定排序；若某一轮不稳定，前面位次的相对顺序会被破坏。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 认为线性排序适用于所有数据
- 忽略关键字范围导致空间爆炸
- 基数排序每轮使用不稳定排序
- 计数排序不处理偏移

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 线性排序的前提：范围、位数和稳定性。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 线性排序 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 面向重排的算法 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 线性排序 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能说明 `n/k/d/r` 的含义，并判断何时不该使用线性排序。
- 能指出一段模型生成代码中最可能出错的地方。
