# Week 01 课程导论与算法分析基础 教师教案

## 本周教学意图

本周不是让学生记住 算法分析 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：共同语言

本周不急着进入某个结构，而是让学生先看到：程序中的对象必须被组织起来，操作才有可分析的边界。

- 选 1 个核心任务让学生课堂手推，建议选择：用同一段代码分别数顺序、循环、二分和递归的执行次数。

## 与前后知识的连接

- 从课程总观念进入：程序不是语句堆砌，而是组织形式和操作规则的组合。
- 导向 Week 02 顺序表：顺序表把元素排成连续线段，位置就是关系，地址计算就是访问能力。

教师讲解时应不断提醒学生：结构复杂度增加，不是为了炫技，而是为了表达更复杂的关系或降低某些操作代价。

## 板书主线

```text
对象：程序 = 数据结构 + 算法、抽象数据类型
关系：从变量、字段和操作开始，建立“结构 + 操作 + 复杂度”的分析框架。
不变量：ADT 是数据对象、数据关系和基本操作的组合。C 语言中常用 struct 表示对象，用函数表示操作，用头文件表达接口边界。
操作：顺序执行、循环扫描、二分缩小
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：同样是 1000 个学生记录，为什么“怎么放”会影响“怎么找、怎么改、怎么排”？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. **顺序执行**：一次完成固定工作量。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(1)。
2. **循环扫描**：元素逐个处理。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
3. **二分缩小**：每轮排除一半。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(log n)。
4. **嵌套枚举**：两层独立循环。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n^2)。
5. **递归分治**：T(n)=aT(n/b)+f(n)。分析时先找会重复执行的语句，再判断重复次数，所以代价为 依递推式而定。

## 本周专属讲法

课堂上不要平均用力，建议抓住下面这条线：

1. 给定 n=8，顺序求和执行 8 次加法，二分查找最多比较 4 次，双重循环比较 28 对元素。
2. 让学生把 n 改为 16，再重新数次数：线性翻倍，对数只多 1 层，平方级增长明显加快。
3. 最后追问：如果函数里有递归，输入规模是 n、n/2 还是调用次数？这一步为后续树、图和排序递归打基础。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 把运行时间和复杂度完全等同
- 只写 O(n)，不说明 n 是什么
- 看到两层循环就机械写 O(n^2)，不看循环范围
- 忽略递归出口

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 用同一段代码分别数顺序、循环、二分和递归的执行次数。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 算法分析 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 共同语言 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 算法分析 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能把一段 C 代码转化为增长函数，并说明输入规模是什么。
- 能指出一段模型生成代码中最可能出错的地方。
