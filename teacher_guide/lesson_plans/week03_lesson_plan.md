# Week 03 线性表之二：链表 教师教案

## 本周教学意图

本周不是让学生记住 链表 的定义，而是让学生看清它在“点 -> 连续线 -> 不连续线 -> 受限线 -> 树 -> 图 -> 索引 -> 重排 -> 综合系统”这条主线中的位置。

阶段定位：不连续的线

它放弃随机访问，换来局部插入删除的灵活性。理解链表，就是理解“关系可以被显式保存”。

## 与前后知识的连接

- 承接 Week 02 顺序表：它是最接近硬件内存的线性结构，随机访问强，但中间插入删除要搬移元素。
- 导向 Week 04 栈：栈、队列和串仍是线性对象，但通过限制操作端点或强化序列匹配，表达更具体的行动规则。

教师讲解时应不断提醒学生：结构复杂度增加，不是为了炫技，而是为了表达更复杂的关系或降低某些操作代价。

## 板书主线

```text
对象：结点、指针域
关系：链表把线性关系从物理相邻转移到指针连接，结点可以散落在内存中。
不变量：链表把线性关系保存在指针中，结点的物理位置不要求连续。操作通常先定位前驱，再改变少量指针。
操作：头插、尾插、按位查找
代价：从字段读写、循环次数和递归层数推出
```

## 两节课教学流程

### 第 1 节：结构是什么

1. 用问题场景引入：为什么链表插入只改两条指针，却仍然常常需要 O(n) 找前驱？
2. 画逻辑结构图，只画对象和关系，暂时不写代码。
3. 把逻辑结构翻译为 C 字段，逐个解释字段责任。
4. 让学生说出本周结构的不变量。
5. 使用交互动画走 2 到 4 步，要求学生记录每一步读写了哪些字段。

### 第 2 节：操作如何维护结构

1. 从 C 示例中找到结构体定义、初始化、核心操作和销毁函数。
2. 手推核心操作，要求学生写出失败条件。
3. 根据移动次数、指针跳转次数、递归层数或边扫描次数推导复杂度。
4. 给一段有缺陷的模型生成代码，让学生指出不变量被破坏的位置。
5. 用 1 道随堂题收束：让学生把操作、复杂度和边界条件写在同一张表中。

## 核心操作讲解脚本

1. **头插**：新结点指向原首元结点，头结点指向新结点。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(1)。
2. **尾插**：若维护尾指针可直接追加。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(1) 或 O(n)。
3. **按位查找**：从头逐步走到第 i 个结点。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
4. **已知前驱插入**：修改两条 next 指针。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(1)。
5. **逆置**：逐结点改变 next 方向。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。

## 本周专属讲法

课堂上不要平均用力，建议抓住下面这条线：

1. 在 10 -> 30 中插入 20，先写 `s->next = p->next`，再写 `p->next = s`。
2. 如果顺序反过来，`p->next` 已经指向 s，原来的 30 入口丢失。
3. 删除 20 时保存待删结点，再让前驱越过它，最后释放内存。每一次赋值都要能在图上画出来。

讲解顺序建议：

1. 先问“操作前结构满足什么不变量”。
2. 再问“为了完成操作，必须读哪些字段”。
3. 接着问“写字段的顺序能否交换”。
4. 最后问“操作后如何证明不变量仍然成立”。

## 常见学生误区

- 把结点值相邻当成内存相邻
- 插入赋值顺序写反
- 删除后继续访问已释放结点
- 忽略空链表和单结点链表

## 课堂活动

- 3 分钟：学生独立画出 5 个元素的小结构图。
- 5 分钟：两人一组互相检查字段含义。
- 8 分钟：全班一起推演 链表插入的两条 `next` 赋值顺序。
- 6 分钟：让 LLM 生成一个错误版本，学生设计能暴露错误的测试。

## LLM 辅助教学提醒

大模型时代，学习 链表 的目的不是和模型比赛背代码，而是拥有判断模型输出的结构能力。学生需要能看出模型有没有维护 不连续的线 的不变量，能不能处理边界，复杂度前提是否成立，测试是否覆盖了最坏情况。知道得越多，越能把模型当成工具，而不是把模型输出当成事实。

建议教师不要简单禁止学生使用大模型，而是要求他们提交“模型输出审查记录”：模型假设了什么、漏了什么边界、复杂度前提是否正确、自己如何修改。

## 形成性评价

学生本周真正掌握的标志：

- 能用一句话说清 链表 保存了什么关系。
- 能画出操作前后结构图。
- 能写出核心操作的 C 风格接口。
- 能画出链表插入/删除前后箭头，并判断是否断链或丢失后继。
- 能指出一段模型生成代码中最可能出错的地方。
