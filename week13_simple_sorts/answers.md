# Week 13 排序算法之一：简单排序 参考答案

> 参考答案用于校准思路，不要求学生逐字一致。程序题重点看接口、边界、复杂度说明和测试证据。

## A. 基础概念参考

1. 本周组织形式应围绕“元素如何保存关系”回答。例如：排序把记录按关键字重新排列。理解排序要同时看比较次数、移动次数、额外空间和稳定性。
2. 图示答案应包含元素、位置或指针关系，并标出操作会改变的位置。树或图题需要标出根、孩子、顶点、边或邻接关系。
3. 函数原型示例可以写成 `int operation(Structure *s, ElemType value);`。关键是区分输入、输出、结构指针和失败返回值。
4. 本周在课程主线中的位置可以概括为：排序把比较、移动、稳定性和额外空间放在一起考察，是结构选择和操作代价的集中训练。

## B. 操作推演参考

5. `插入排序` 的关键动作是：维护已排序前缀，将新元素插入合适位置。推演时必须写出读字段、写字段和不变量。
6. 失败路径至少包含空结构、满结构、越界、查找失败、重复关键字或非法输入中的一种。
7. 三段式证明示例：操作前结构满足 简单排序 的定义；操作中只修改必要字段；操作后对象关系仍然符合定义。

## C. 代码阅读与测试参考

8. 初始化函数负责建立合法空状态；核心修改函数负责改变结构字段；输出函数用于观察状态，不应替代测试。
9. 合格测试至少包含正常路径、边界路径和失败路径。例如空结构删除应失败，查找不存在元素应返回 -1 或 NULL，容量不足应扩容或拒绝。
10. C 语言细节通常落在数组下标、指针重连、动态内存、字符串结束符、递归栈或结构体字段初始化。

## D. 复杂度或设计评审参考

- 插入排序：O(n^2)，近有序时好。推导依据：维护已排序前缀，将新元素插入合适位置。
- 选择排序：O(n^2)，移动少。推导依据：每轮选择最小元素放到前部。
- 冒泡排序：O(n^2)。推导依据：相邻逆序则交换，大元素逐步后移。
- 稳定性判断：依算法而定。推导依据：相等关键字是否保持相对次序。
- 提前终止：最好 O(n)。推导依据：一趟无交换即可停止冒泡。

最坏情况需要明确输入规模。例如元素个数 `n`、顶点数 `V`、边数 `E`、模式串长度 `m` 或关键字范围 `k`。不能只写大 O 结论。

## E. 错误代码诊断参考

题目：冒泡排序提前停止标志未重置

代码：

```c
int swapped = 0; for (...) { if (...) swapped = 1; }
```

问题：每一趟开始都要重置 swapped，否则无法判断本趟是否已有序。

评分时重点看学生是否能说出：

1. 哪个不变量被破坏。
2. 哪个最小输入可以触发错误。
3. 修正方案是否真的覆盖边界。

## F. 本周专属训练参考

本周专属训练重点看学生是否完成了下面这些能力：

- 同一数组分别手推三种排序前两轮
- 用带编号相等关键字判断稳定性
- 让 LLM 给出排序代码并补重复关键字测试

参考答案不要求形式一致，但必须体现本周核心抓手：简单排序的比较、移动和稳定性。

## G. 设计与 AI 审查参考

参考方向：稳定性要用带标签的重复关键字跟踪相对顺序。插入排序通常稳定，选择排序通常不稳定，冒泡排序在只交换逆序相邻元素时稳定。性能比较要同时记录比较次数和移动/交换次数。
