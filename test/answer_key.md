# 测试题参考答案与评分尺度

## 随堂测试

随堂测试以短答案为主，参考尺度如下：

- 组织形式说清楚：2 分。
- 操作过程能落到字段或位置变化：3 分。
- 复杂度结论正确且说明输入规模：3 分。
- 边界情况合理：2 分。

## 编程作业

- 正确性：40%。核心操作能通过正常和边界测试。
- 结构设计：20%。ADT 接口清晰，模块职责明确。
- 复杂度分析：15%。能说明关键操作的时间和空间代价。
- C 语言质量：15%。处理内存、越界、错误返回和命名。
- LLM 使用记录：10%。能说明模型输出如何被验证和修正。

## 常见扣分点

1. 只给代码，不说明结构组织形式。
2. 操作函数没有失败返回值。
3. 动态内存申请后没有释放。
4. 把平均复杂度写成最坏复杂度，或没有说明前提。
5. 直接粘贴 LLM 代码，无法解释字段和边界。
