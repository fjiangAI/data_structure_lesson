# 实验作业说明

本文件夹提供跨周实验模板。每个实验都要求学生提交代码、测试和简短说明，强调“能运行”之外的结构理解。

每个 Lab 同时提供两种材料：

- `labXX_*.md`：教师布置作业时可直接引用的完整要求。
- `labXX_*/`：学生可运行 ADT starter，包含 `starter/*.h`、`starter/*.c`、`tests/public_main.c`、`expected_output.txt` 和 `tests/test_lab.py`。
- [lab_testing_guide.md](lab_testing_guide.md)：测试脚本、统一运行器和 hidden tests 建议。
- [labs.json](labs.json)：Lab 元数据清单，便于后续接入课程平台或自动批改系统。

推荐节奏：

- Lab 01：Week 2-3 后布置，线性表。
- Lab 02：Week 4-6 后布置，栈、队列、串。
- Lab 03：Week 7-9 后布置，树、AVL、堆。
- Lab 04：Week 10-12 后布置，图、查找、哈希。
- Lab 05：Week 13-15 后布置，排序算法。
- Lab 06：Week 16 作为综合课程设计。

统一提交物：

1. 源代码和编译命令。
2. 至少 6 个测试用例，其中 2 个必须是边界情况，1 个必须是能暴露常见错误的反例。
3. 结构不变量说明：写清核心字段含义，以及插入、删除、查找或遍历后哪些关系必须保持。
4. 复杂度说明：写清输入规模、最好/最坏或平均前提，不把 public test 的规模当作复杂度依据。
5. 测试证据：public test 输出、自造边界测试输出，以及一个失败路径或异常输入的处理说明。
6. AI 使用记录，格式参考 [../AI_LEARNING_GUIDE.md](../AI_LEARNING_GUIDE.md)。记录中必须包含至少一个人工核查点。

高阶提交物（教师可按班级水平选用）：

- 反例构造：说明某个错误实现会在哪个输入上失败，并画出失败前后的结构状态。
- 性能实验：对至少 3 种输入分布记录比较次数、移动次数、探测次数或运行时间，并解释一个反常现象。
- 代码审查：给出一段 LLM 或同伴生成代码，指出它维护了哪些不变量、遗漏了哪些边界。
