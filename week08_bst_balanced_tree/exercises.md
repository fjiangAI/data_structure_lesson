# Week 08 二叉搜索树与平衡树基础 练习题

## A. 基础必做：概念与结构

1. 用不超过 120 字说明本周结构保存了什么“对象”和什么“关系”。
2. 画出一个包含 5 个元素的小例子，标出关键字段、指针、下标、父子关系或邻接关系。
3. 写出本周最核心 ADT 的 C 风格接口，至少包含初始化、核心操作、销毁或清理。
4. 解释本周结构在课程主线中的位置：它相对于上一类结构增加了什么表达能力，付出了什么代价？

## B. 手推必做：操作过程

5. 手工执行 `BST 查找`，逐步写出每一步读了哪些字段、写了哪些字段。
6. 对 `BST 插入` 给出正常路径和失败路径各一个例子。
7. 选择一个操作，写出“操作前不变量、操作步骤、操作后不变量仍成立”的三段式说明。

## C. 编程必做：代码阅读与测试

8. 阅读 [examples/avl_tree.c](examples/avl_tree.c)，指出结构体定义、初始化函数、核心修改函数和输出函数。
9. 给示例代码补 3 个测试：正常情况、边界情况、错误输入或失败路径。
10. 找出示例代码中最依赖 C 语言细节的一处，例如指针、数组下标、动态内存或字符串结束符。

## D. 复杂度与证明（挑战）

11. 对下列操作分别说明输入规模和复杂度来源：

    - BST 查找：平均 O(log n)，最坏 O(n)，关键动作：按关键字大小选择左右子树
    - BST 插入：平均 O(log n)，最坏 O(n)，关键动作：查找失败位置成为新叶子
    - BST 删除：平均 O(log n)，最坏 O(n)，关键动作：叶子、单孩子、双孩子分别处理
    - AVL 旋转：O(1)，关键动作：局部改变父子关系恢复高度
    - AVL 插入：最坏 O(log n)，关键动作：沿递归返回路径更新高度并旋转

12. 构造一个最坏情况输入，并说明它为什么触发最坏复杂度。
13. 如果输入规模扩大 100 倍，本周结构最先暴露的瓶颈是什么？

## E. 错误代码诊断（必做）

下面这段代码很像 LLM 生成的简化实现，请指出 bug、给出触发 bug 的输入，并写出修正思路。

```c
if (key < root->key) root = root->right; else root = root->left;
```

诊断要求：

1. 说明它破坏了哪个结构不变量。
2. 写一个最小反例。
3. 给出修正后的关键语句或伪代码。

## G. 设计与 AI 审查（拓展）

1. 为 LL、RR、LR、RL 四种 AVL 失衡各构造一个最小插入序列，并画出旋转前后的中序序列。
2. 让 LLM 写 AVL 旋转代码，检查它是否更新高度、是否保持 BST 中序有序、是否处理子树接回。
3. 构造一个普通 BST 最坏退化输入，比较未平衡和 AVL 的树高变化。
4. 提交证据：四种旋转图、错误旋转代码风险、树高对比表。
