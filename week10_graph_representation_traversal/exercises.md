# Week 10 图的表示与遍历 练习题

## A. 基础必做：概念与结构

1. 用不超过 120 字说明本周结构保存了什么“对象”和什么“关系”。
2. 画出一个包含 5 个元素的小例子，标出关键字段、指针、下标、父子关系或邻接关系。
3. 写出本周最核心 ADT 的 C 风格接口，至少包含初始化、核心操作、销毁或清理。
4. 解释本周结构在课程主线中的位置：它相对于上一类结构增加了什么表达能力，付出了什么代价？

## B. 手推必做：操作过程

5. 手工执行 `添加边`，逐步写出每一步读了哪些字段、写了哪些字段。
6. 对 `判断边存在` 给出正常路径和失败路径各一个例子。
7. 选择一个操作，写出“操作前不变量、操作步骤、操作后不变量仍成立”的三段式说明。

## C. 编程必做：代码阅读与测试

8. 阅读 [examples/graph_traversal.c](examples/graph_traversal.c)，指出结构体定义、初始化函数、核心修改函数和输出函数。
9. 给示例代码补 3 个测试：正常情况、边界情况、错误输入或失败路径。
10. 找出示例代码中最依赖 C 语言细节的一处，例如指针、数组下标、动态内存或字符串结束符。

## D. 复杂度与证明（挑战）

11. 对下列操作分别说明输入规模和复杂度来源：

    - 添加边：O(1)，关键动作：矩阵赋值或链表头插
    - 判断边存在：O(1) 或 O(deg)，关键动作：矩阵直接读，邻接表扫描
    - DFS：O(V+E)，关键动作：沿一条路径尽量深入
    - BFS：O(V+E)，关键动作：按距离层次扩展
    - 连通分量：O(V+E)，关键动作：多次从未访问顶点启动遍历

12. 构造一个最坏情况输入，并说明它为什么触发最坏复杂度。
13. 如果输入规模扩大 100 倍，本周结构最先暴露的瓶颈是什么？

## E. 错误代码诊断（必做）

下面这段代码很像 LLM 生成的简化实现，请指出 bug、给出触发 bug 的输入，并写出修正思路。

```c
enqueue(v); /* visited[v] later */
```

诊断要求：

1. 说明它破坏了哪个结构不变量。
2. 写一个最小反例。
3. 给出修正后的关键语句或伪代码。

## F. 本周专属训练

1. 手推一张 5 顶点图的 DFS/BFS
2. 设计有环图测试
3. 让 LLM 生成 BFS 并检查标记时机

## G. 设计与 AI 审查（拓展）

1. 比较邻接矩阵和邻接表在稠密图、稀疏图上的空间代价，并给出适用场景。
2. 构造一个含环图，比较 BFS 在“入队时标记”和“出队时标记”下的队列变化。
3. 让 LLM 生成 DFS/BFS，检查它是否处理非连通图、是否正确初始化 visited。
4. 提交证据：图表示选择理由、队列/栈状态表、模型遗漏的遍历边界。
