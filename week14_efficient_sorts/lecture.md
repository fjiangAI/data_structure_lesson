# Week 14 排序算法之二：快速、归并与堆排序

## 本周定位

掌握快速排序、归并排序、堆排序的核心思想，比较它们在时间、空间和稳定性上的取舍。

本课程把数据结构看作“数据的组织形式”，把算法看作“作用在组织形式上的操作过程”。当组织形式变了，同一个操作的代价、边界条件和程序写法都会变化。


## 结构演进中的位置

本周处在课程主线的“面向重排的算法”阶段。

排序不是新结构，而是在数组或记录序列上重建顺序关系。

排序把比较、移动、稳定性和额外空间放在一起考察，是结构选择和操作代价的集中训练。

- 下一周的 线性排序 会继续推进：排序不是新结构，而是在数组或记录序列上重建顺序关系。

## 大模型时代为什么还要学这一章

高效排序是大模型最容易讲得过于笼统的章节：它可能只说快排平均 O(n log n)，却不说明 partition 不变量和最坏退化；也可能混淆归并、堆排的空间与稳定性。本章训练学生把单次划分、递归树和实现取舍分开看。这样才能判断模型推荐的排序算法是否匹配输入分布和工程约束。

## 核心概念

高效排序的核心是减少无效比较。快速排序靠划分，归并排序靠有序子序列合并，堆排序靠优先队列结构。

讲课时建议始终抓住三个问题：

- 数据对象是什么：元素、结点、记录、顶点还是字符。
- 关系如何保存：连续位置、指针、父子关系、邻接关系、关键字映射还是有序关系。
- 操作如何改变结构：插入、删除、查找、遍历、排序、合并或旋转到底修改了哪些字段。


## 从问题出发理解本周结构

本周可以从这个具体问题进入：比较快速排序、归并排序和堆排序的核心状态变化。

学习时不要先背定义，而要先回答：如果只用最朴素的数组、循环或指针，会在哪个操作上变慢、变乱或容易出错？数据结构的价值就在于把对象之间的关系稳定地组织起来，让操作可以按照可预测的方式执行。

## 结构不变量

高效排序都把 O(n^2) 的无效比较压缩掉：快排靠划分，归并靠有序子段合并，堆排序靠堆序性质反复选出最大值。

不变量是实现数据结构时最重要的检查标准。每个操作执行前不变量应当成立，操作执行后也必须重新成立。如果程序出错，通常就是某一步破坏了不变量，例如顺序表的 `size` 与实际元素不一致，链表断链，队列 `front/rear` 含义混乱，或者树的有序性被破坏。

## 存储表示拆解

高效排序通过分治或结构约束减少比较。快速排序维护划分边界，归并排序维护有序子段，堆排序维护堆序。

把逻辑结构翻译成 C 语言时，重点不是写出一个 `struct`，而是解释每个字段承担什么责任。字段一旦设计错误，后面的操作会变得复杂甚至无法保证正确。

## 核心操作推演

1. **快速排序划分**：单次扫描把小于枢轴的元素放左侧。分析时先找会重复执行的语句，再判断重复次数，所以代价为 单次 O(n)，整体平均 O(n log n)。
2. **归并**：两个有序段线性合并。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n)。
3. **堆排序**：建堆后反复交换堆顶与末尾。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(n log n)。
4. **递归深度**：影响栈空间和最坏情况。分析时先找会重复执行的语句，再判断重复次数，所以代价为 O(log n) 到 O(n)。
5. **稳定性**：归并可稳定，快排和堆排序通常不稳定。这是性质判断题，要用相等关键字、移动方式或结构不变量来证明。

本周交互演示中的状态变化如下：

1. **快速排序：选择 pivot=10**：快速排序选择 10 为枢轴，扫描前面的元素，把小于等于 10 的元素交换到左侧。
2. **快速排序：发现 3**：扫描到 3 时，它小于 pivot，于是和小区间之后的第一个元素交换。
3. **快速排序：发现 9**：继续扫描到 9，也放到左区间。此时左区间是 [3,9]。
4. **快速排序：pivot 归位**：扫描结束后把 pivot 放到左右区间中间。10 的最终位置已经确定，递归处理两侧。
5. **归并排序：拆分为有序段**：归并排序先递归拆分，直到小段有序，再开始合并相邻有序段。
6. **归并排序：比较段首**：合并时只比较两个有序段的段首。9 更小，所以它进入辅助数组。
7. **归并排序：完成合并**：每个元素只被搬入辅助数组一次，因此一次合并是 O(n)，整体是 O(n log n)。
8. **堆排序：建最大堆**：堆排序先把数组调整为最大堆。堆只保证父结点不小于孩子，不保证整体有序。
9. **堆排序：堆顶交换到末尾**：把堆顶最大值 82 与末尾交换，末尾成为有序区，然后对根执行下滤。
10. **堆排序：下滤恢复堆**：10 破坏了堆序，通过下滤与较大的孩子交换，恢复剩余区间的最大堆。
11. **堆排序：有序区扩大**：重复取堆顶，右侧有序区不断扩大。堆排序时间 O(n log n)，额外空间 O(1)。

建议学生在纸上跟着演示重画结构图。每一步都要标出“读了哪些字段、写了哪些字段、哪些关系保持不变”。

## 复杂度推导方法

复杂度不是结论表，而是从操作过程推出来的。分析时按下面顺序：

1. 明确输入规模，例如元素个数 `n`、模式串长度 `m`、顶点数 `V`、边数 `E` 或关键字范围 `k`。
2. 找到最内层重复执行的基本动作，例如比较、赋值、指针跳转、交换或松弛。
3. 说明这个动作最多执行多少次。
4. 区分最好、平均和最坏情况，尤其是查找、哈希和快速排序。
5. 写出额外空间来自哪里，例如辅助数组、队列、栈或递归调用栈。

## 结合代码理解思想

快速排序代码重点看 partition 的循环不变量：扫描到 j 之前，左侧区间已经不大于 pivot。

阅读代码时建议按“结构定义 -> 初始化 -> 核心操作 -> 边界处理 -> 释放资源”的顺序。不要从 `main` 开始只看输出结果；数据结构课更关心状态如何被维护。



## 本周完整推导例题

教师可以把下面例题作为课堂主线，学生应能在纸上复现。

1. partition 对一个长度为 n 的区间只扫描一次，因此单次代价是 O(n)。
2. 若每次划分接近均衡，递归树高度约为 log n，每层总扫描 O(n)，整体平均 O(n log n)。
3. 若 pivot 总是最小或最大，递归高度退化到 n，整体最坏 O(n^2)。归并稳定但要 O(n) 辅助空间，堆排 O(1) 额外空间但通常不稳定。


## 高阶诊断：partition 的循环不变量

快速排序的难点不在递归框架，而在 partition 是否维护了清晰的不变量。下面这种写法在大量重复关键字时容易卡住：

```c
while (i < j) {
    while (a[i] < pivot) ++i;
    while (a[j] > pivot) --j;
    if (i < j) swap(&a[i], &a[j]);  /* 缺少交换后的推进 */
}
```

当 `a[i] == pivot` 且 `a[j] == pivot` 时，交换不会改变数组，`i` 和 `j` 也没有推进，循环可能重复同一状态。课堂上应让学生用 `[5,5,5]` 或 `[3,5,5,2]` 手推，明确每轮后左区间、右区间和未知区间分别是什么。

可用追问：单次 partition 是 O(n)，为什么整体 quicksort 最坏仍可能是 O(n^2)？

## C 语言表示

示例代码见 [examples/efficient_sorts.c](examples/efficient_sorts.c)。课堂建议先读结构体定义，再读初始化函数，然后读最能体现结构特点的操作函数。

一个可靠的 C 语言实现通常包含：

- 明确的结构体定义，字段名能反映含义。
- 初始化和销毁函数，避免未初始化状态和内存泄漏。
- 操作函数返回成功或失败，调用者能处理异常情况。
- 对数组下标、空指针、容量和边界位置做检查。
- 用小规模样例打印结构状态，帮助观察每一步变化。

## 关键操作与复杂度

| 操作 | 组织方式/关键动作 | 复杂度 |
|---|---|---|
| 快速排序划分 | 单次扫描把小于枢轴的元素放左侧 | 单次 O(n)，整体平均 O(n log n) |
| 归并 | 两个有序段线性合并 | O(n) |
| 堆排序 | 建堆后反复交换堆顶与末尾 | O(n log n) |
| 递归深度 | 影响栈空间和最坏情况 | O(log n) 到 O(n) |
| 稳定性 | 归并可稳定，快排和堆排序通常不稳定 | 概念判断 |

复杂度分析不要只背结论。要说明输入规模是什么，循环次数或递归层数如何随输入增长，额外空间来自数组、递归栈还是辅助结构。

## 常见误区

- 把 partition 写成 O(n log n)
- 忽略重复关键字导致极端不平衡
- 把归并的单次合并和整体复杂度混淆
- 认为堆排后堆数组一开始就是有序的
